# GitLab MR 作成

現在のブランチからマージリクエストを作成し、メタデー���を設定する。

## ワークフロー

1. **ブランチコンテキストの確認**
   - 現在のブランチ名を `git branch --show-current` で取得する。
   - デフォルトブランチ（main/master等）で作業中の場合は、MR作成できない旨を伝える。
   - 未コミットの変更がある場合は、先にコミットを推奨する。
   - 未プッシュのコミットがある場合は `git push -u origin <branch>` を実行する。

2. **既存MRの確認**
   - `glab_mr_for` を実行して、現在のブランチに既存のMRがないか確認する。
   - 既存MRがある場合：
     - MRのURL、ステータスを報告する。
     - ユーザーに更新を提案する（`glab_mr_update` でタイトル・説明・ラベル等の変更）。
     - 新規作成は行わない。

3. **MR内容の生成**
   - `git log --oneline <target_branch>..HEAD` でコミット履歴を取得する。
   - `git diff <target_branch>...HEAD --stat` で変更ファイルの統計を取得する。
   - コミット履歴と変更内容からMRタイトルと説明を生成する：
     - **タイトル**: 変更の要旨を簡潔に（コミットが1つの場合はそのメッセージを使用）。
     - **説明**: 変更の概要、影響範囲、テスト方法を含める。
   - ブランチ名からイシュー番号を自動検出する：
     - `feature/123-add-login` → `Closes #123`
     - `fix/456-crash-on-start` → `Fixes #456`
     - `issue-789-update-docs` → `Related to #789`
   - 検出されたイシュ��番号は説明文に自動で含める。

4. **メタデータの設定**
   - ユーザーの指示に基づき以下を設定する：
     - **ラベル**: `glab_label_list` で利用可能なラベルを確認��、適切なものを選択。
     - **マイルストーン**: `glab_milestone_list` で利用可能なマイルストーンを確認。
     - **レビュアー**: ユ��ザー指定またはコードオーナーから選択。
     - **担当者**: ユーザー指定または自分自身をデフォルトで設定。
   - ユーザーが特に指定しない場合は、最小限のメタデータで作成する。

5. **MRの作成**
   - `glab_mr_create` を以下のフラグで実行する：
     - `--title "<title>"` - MRタイトル
     - `--description "<description>"` - MR説明
     - `--target-branch "<branch>"` - マージ先ブランチ（デフォルトはリポジトリのデ���ォルトブランチ）
     - `--label "<label>"` - ラベル（複数指定可）
     - `--milestone "<milestone>"` - マイルストーン
     - `--reviewer "<username>"` - レビュアー（複数指定可）
     - `--assignee "<username>"` - 担当者（複数指定可）
     - `--draft` - ドラフトMRとして作成する場合
   - ユーザーがドラフトを要求した場合、または作業が未完了の場合は `--draft` を付与する。

6. **結果の報告**
   - 作成されたMRのURL、IID、タイトルを報告する。
   - 設定されたメタデータ（ラベル、マイルストーン、レビュアー、担当者）を確認する。
   - 次のステップを提案する：
     - CIパイプラインの確認 → `$gitlab-ci`
     - レビュー依頼の送信完了を伝える

## イシュー番号の自動検出パターン

ブランチ名から以下のパターンでイシュー番号を検出する：

| ブランチ名パターン | 検出結果 |
|-------------------|---------|
| `feature/<IID>-<slug>` | `Closes #<IID>` |
| `fix/<IID>-<slug>` | `Fixes #<IID>` |
| `bugfix/<IID>-<slug>` | `Fixes #<IID>` |
| `hotfix/<IID>-<slug>` | `Fixes #<IID>` |
| `issue-<IID>-<slug>` | `Related to #<IID>` |
| `<IID>-<slug>` | `Related to #<IID>` |
