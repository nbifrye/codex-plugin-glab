# GitLab イシュー取得

イシューの本文と全コメント（ノート）を取得し、構造化された形式で提示する。

## ワークフロー

1. **イシューの特定**
   - ユーザーがイシュー番号を提供した場合、それを直接使用する。
   - URLが提供された場合、プロジェクトパスとIIDを抽出する。
   - ローカルのgitリモートまたはユーザー入力からプロジェクトパスを特定する。

2. **イシュー本文の取得**
   - `glab_issue_view <IID> --output json` を実行してイシューの構造化データを取得する：
     - タイトル、説明（本文）、作成者、ラベル、マイルストーン
     - 担当者、作成日時、更新日時、状態（open/closed）
     - 関連リンク（`linked_issues`, `linked_mr`）
   - 説明が長い場合もすべて取得する。

3. **コメント（ノート）の取得**
   - `glab_issue_view <IID> --comments` で人間が読みやすい形式のコメント概要を取得する。
   - 構造化データが必要な場合、または全コメントのページネーションが必要な場合：
     - `glab_api` GET `projects/<encoded_project>/issues/<iid>/notes` を使用する。
     - ページネーション: `per_page`（最大100）と `page` パラメータで全ページを取得する。
     - 大量のコメントがある場合は `--paginate` フラグで自動ページネーションを使用する。
   - 詳細は `references/gitlab-issue-notes-api.md` を参照すること。

4. **システムログの取得（任意）**
   - ユーザーがイシューの変更履歴を求めた場合、`glab_issue_view <IID> --system_logs` でシステムイベントを取得する。
   - ラベル変更、担当者変更、マイルストーン変更、状態変更等のイベントが含まれる。

5. **情報の構造化**
   - 取得した情報を以下の形式で時系列に整理して提示する：
     - **メタデータ**: タイトル、状態、ラベル、マイルストーン、担当者、作成者、日時
     - **本文**: イシューの説明（description）全文
     - **コメント一覧**: 各コメントの投稿者、日時、内容を時系列で表示
     - **関連アイテム**: 関連MR、関連イシューのリンク
   - システムノート（`system: true`）とユーザーコメントを区別して表示する。

6. **要約の提供**
   - ユーザーが求めた場合、イシューの議論の要約を提供する：
     - 主要な論点と結論
     - 未解決の質問や懸案事項
     - 次のアクションアイテム

## コメント取得の詳細手順

構造化コメントデータが必要な場合、以下の手順に従う：

1. プロジェクトパスをURLエンコードする（`/` → `%2F`）。

2. `glab_api` を呼び出す。**Go テンプレートレンダリングによる値の破損を防ぐため、`raw_field`（`field` ではなく）を使用する：**

   ```
   args: ["projects/<encoded_project>/issues/<iid>/notes"]
   flags:
     method: "GET"
     raw_field:
       - "per_page=100"
       - "sort=asc"
       - "order_by=created_at"
   ```

3. レスポンスの各ノートから以下を抽出する：
   - `id` - ノートID
   - `body` - コメント本文
   - `author.username` - 投稿者
   - `created_at` - 投稿日時
   - `system` - システムノートかどうか（true/false）
   - `noteable_iid` - 関連イシューIID

4. 全ページを取得するまでページネーションを繰り返す。

## 出力

イシューのメタデータ、本文、全コメントを構造化された形式で提示する。コメント数が多い場合は合計数も報告する。
